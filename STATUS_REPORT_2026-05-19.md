**Defect:** `#SmartPROHubHubHubHubHub` (five `Hub` repetitions appended)

### Root cause

The old `_sanitise_hashtags` in `generator.py` looped through brand variants and applied `re.sub` per variant. Each variant (e.g. `#SmartPro`) matched the `#SmartPRO` prefix of the canonical replacement `#SmartPROHub`, leaving a trailing `Hub` to compound on the next iteration. Five matching variants produced five extra `Hub`s.

Commit `ad71e2a` (Tue May 19 19:00 +0400) fixed this by switching to a single regex with word-boundary lookarounds. Posts generated before that commit, however, were never re-sanitized — sanitization runs only at generation time.

## Repo hygiene flag — CRLF/LF line-ending churn

`git status` shows 36 modified files in your working copy. A diff stat reveals 4067 insertions vs 4067 deletions — every "modification" is line-ending churn, not real content change. The repo's `.gitattributes` only pins `docs/index.html` and `dashboard.py` to LF; everything else picks up CRLF on Windows checkouts.

If you commit this state, you'll create a noisy diff that obscures real changes and may produce merge conflicts on future rebases.

**Fix:** add to `.gitattributes`:

```
* text=auto eol=lf
*.json text eol=lf
*.py   text eol=lf
*.md   text eol=lf
```

then run `git add --renormalize .` once.
